# Release Notes: v5.0.0

**Date**: May 28, 2026  
**Type**: Major Release  

Welcome to the **Windows Optimization Framework v5.0**! This is the most significant update in the project's history. We have shifted from a collection of raw PowerShell scripts to a cohesive, unified, visually stunning desktop application experience, without sacrificing our core principles of transparency, safety, and modularity.

## What's New?

### 1. The WPF Dashboard
The heart of v5.0 is the new `Dashboard.xaml` GUI. It is completely native, relying entirely on PowerShell and Windows Presentation Foundation (WPF). There are no opaque compiled binaries. 
- You can now apply **Max Performance**, **Balanced Creator**, or **Enterprise Compliance** profiles with a single click.
- Real-time **Observability** metrics tick seamlessly in the background using blazing fast `.NET Performance Counters` without freezing the UI.

### 2. The Rollback Engine
We have fully eliminated the fear of "breaking your installation." 
Before any profile runs, the `SnapshotEngine.psm1` takes a deep snapshot of your Registry and Services, exporting it as a JSON manifest. If you don't like a tweak, head to the **Rollback Engine** tab in the Dashboard, select the timestamp, and click restore. It's that simple.

### 3. Zero-Friction Setup
We've made it incredibly easy to start. Just extract the ZIP, double-click `Start.bat`, and follow the prompt. The framework will offer to create a native Windows System Restore point and place a shortcut on your Desktop automatically.

## Known Issues
- Currently, the Rollback Engine only captures Registry and Service states. File deletions (e.g., removing OneDrive) are not inherently reversible via the JSON manifest. We recommend allowing `Start.bat` to create a standard Windows System Restore point to cover edge cases.
